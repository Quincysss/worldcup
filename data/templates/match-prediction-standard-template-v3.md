# 单场量化预测标准模板 v3

比赛：`YYYY-MM-DD 球队A vs 球队B`  
阶段：`小组/轮次`  
竞彩编号：`如适用`  
预测状态：`discussion_only_red_team_hold / red_team_pass / temporary`  
投注状态：`discussion_only / hold_not_executable`  
红队状态：`pending / hold / revise / pass`

## 结论

用 2-4 段说明模型结论、最高概率方向、最高单比分、主要风险和投注边界。必须包含：

| 结果 | 概率 |
| --- | ---: |
| 球队A胜 | `%` |
| 平局 | `%` |
| 球队B胜 | `%` |

主比分倾向：`x-y`。  
次选比分：`a-b、c-d、e-f、g-h`。  
方向表述：只写模型方向，不写主推、稳胆、重注、包赢。

## 前两轮解读

分别说明两队已完成比赛的事实、战术表现、模型需要继承或修正的信号。第三轮必须写清楚小组形势、排名目标、平局价值、净胜球目标和同时开球影响。

## 预计首发名单

状态：`confirmed / probable_not_official / uncertain / conflicting`。  
捕获时间：`YYYY-MM-DD HH:mm TZ`。  
更新闸门：T-75 官方首发发布后必须重核；若核心轮换超过 2 人或阵型变化，需重跑 xG 和泊松矩阵。

### 球队A预计首发

预计阵型：`4-3-3`。  
首发置信度：`high / medium_high / medium / medium_low / low`。  
主要来源：`source names`。

注意：T-75 官方首发未公布前，首发置信度通常不得高于 `medium`；若写到 `medium_high` 或 `high`，必须说明已接近官方/多源完全一致且无轮换动机。

| 位置 | 球员 | 角色说明 |
| --- | --- | --- |
| 门将 | 中文名 English Name | 一门/出球/扑救角色 |
| 后卫 | 中文名 English Name | 防守、推进或定位球角色 |
| 中场 | 中文名 English Name | 覆盖、组织、前插或压迫角色 |
| 前锋 | 中文名 English Name | 支点、冲刺、终结或牵制角色 |

轮换候选：

| 球员 | 触发条件 |
| --- | --- |
| 中文名 English Name | 何种战术/伤停/轮换条件下首发或提前登场 |

首发对模型的影响：

- 若核心球员A首发/缺席，xG、节奏、定位球或转换因子如何变化。
- 若阵型从主口径改为备选口径，哪些概率需要重跑。
- 若轮换明显，第三轮动机和节奏如何调整。

### 球队B预计首发

同球队A格式。不得省略首发置信度、轮换候选、模型影响和来源。

### 首发缺口与冲突

- 官方首发是否公布。
- 不同来源在哪些位置、阵型、伤停或轮换口径上冲突。
- 哪些变化会触发重新跑模型。

## 权重评分

列出因子权重、两队评分和简短说明。第三轮比赛必须单列第三轮情境或把其权重说明清楚。

## xG 与概率链条

必须包括：最终 xG、原始泊松胜平负、赔率校准后最终概率、置信区间、大小球、BTTS。

## 泊松比分矩阵

必须输出 0-0 到 5-5 矩阵、tail、Top 5 比分。

## 赔率隐含概率校准

赔率只做校准和风险观察。必须写来源、捕获时间、去水隐含概率、模型差值、市场是否过热。

## 战术对位

说明两队关键进攻路径、防守风险、替补触发、领先/落后时的比赛状态变化。战术结论必须能映射到模型因子。

## 红队风险

写红队状态、主要风险、是否允许发布、投注闸门。若红队为 `hold` 或 `revise`，不得输出可执行投注建议。

## 文件索引

列出结构化 JSON、数据包、红队审查、竞彩风险、来源快照和线程输出路径。

## 来源

列出所有时效字段来源 URL、本地快照路径和捕获时间。来源冲突必须保留，不强行拍板。
