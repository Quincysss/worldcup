# G/H组第一轮投注流程纠偏

phase: betting_process_review  
status: active_rule_update  
updated_at: 2026-06-17T15:30:00+08:00  
scope: 针对 G/H 组第一轮预测失误，尤其是“比利时胜”推荐失败，修正后续世界杯预测和体彩建议流程。

## 为什么必须单独写这份

比利时 1-1 埃及不是一个可以轻轻带过的失误。用户按“比利时胜”方向买输了，而赛前材料其实已经识别出比利时防转换、埃及反击、Salah/Marmoush 状态、Lukaku/De Bruyne 体能和即时盘口未锁等风险。

真正的问题是：风险写进了分析，但没有进入最终投注决策。

以后这类情况必须触发流程闸门，而不是继续用“中等偏低置信度”包装一个热门胜方向。

## 红队结论

| 问题 | 严重度 | 说明 |
| --- | --- | --- |
| 阻断条件失效 | 高 | 红队 `hold/revise` 后仍输出主胜方向 |
| 赔率快照不足 | 高 | 没有完整盘口，却写条件式投注判断 |
| 强队名气偏差 | 高 | 比利时、西班牙、乌拉圭都被默认小胜 |
| 平局保护不足 | 高 | G/H 四场全平，赛前却四场都给热门胜 |
| 让球保护不足 | 中高 | 文字提醒深盘风险，但投注口径仍偏热门 |
| 风险与概率脱节 | 高 | 风险在正文，概率和主推没有相应下调 |

## 立即生效的投注输出闸门

以下任一条件满足时，禁止输出“主推”“主方向”“可小注”“稳胆”“胜胆”等语言：

1. 预测文件状态仍是 `pre_match_partial`。
2. 红队结论为 `hold` 或 `revise`，且关键变量未复核。
3. T-75m 首发未确认，且核心球员状态会显著影响比赛。
4. 没有完整可复核赔率快照。
5. 热门队存在明显名气溢价，对手有低位纪律、速度反击或定位球路径。
6. 预测比分集中在 1-0、1-1、2-1 这类窄比分区间，但让球盘口较深。

触发闸门后的允许输出：

- 风险排序。
- 情景分析。
- 等首发/等盘口。
- 不下注。
- 平局/受让保护提示。

触发闸门后的禁止输出：

- 单选主胜。
- 深让球方向。
- 串关胆材。
- “小注可考虑”这类容易被理解为可执行建议的语言。

## 后续体彩建议模板

每场给体彩建议前必须先写四行：

1. `临场状态`: 已确认 / 未确认。
2. `盘口快照`: 已捕获 / 未捕获。
3. `红队闸门`: 通过 / 未通过。
4. `购买等级`: 可讨论 / 只做保护 / 跳过。

如果 `红队闸门=未通过`，则最终结论只能是：

- 跳过；
- 或只做保护，不给主推；
- 或等待 T-75m 首发和盘口后重算。

## 比利时 vs 埃及的正确赛前输出应该是什么

错误输出：

- 比利时胜方向。
- 比利时 2-1。
- 比利时不败 / 小胜方向。

更合理输出：

- 比利时过程优势存在，但三分不稳。
- 平局保护必须保留。
- 若只能买胜平负，不建议把比利时胜当胆。
- 若让球为比利时 -1，倾向让平/让负保护，不碰让胜。
- 若首发和盘口不能临场复核，跳过。

## 模型参数修正

| 参数 | 修改 |
| --- | --- |
| draw_baseline_round1 | 上调，尤其是热门对低位队 |
| favorite_conversion | 下调，过程优势不得自动转为胜率 |
| market_name_bias | 新增硬惩罚项 |
| low_block_resistance | 上调，弱队防守执行可直接提高平局 |
| transition_threat | 从尾部风险升为主变量 |
| lineup_gate | 未确认核心首发时，不给可执行投注建议 |
| odds_snapshot_gate | 无完整赔率时，不给盘口建议 |

## 用户侧风险提示固定语

以后在涉及投注的回答中，必须加入足够明确的风险提示，尤其是：

- 以下判断不是临场定稿，若 T-75m 首发、伤停或盘口与预期不符，本建议自动失效。
- 热门有过程优势，不等于热门有稳定三分优势。
- 若热门存在名气溢价或深盘升温，默认不追。
- 平局和受让保护必须保留；如果做不到保护，宁可不做。
- 核心球员状态未确认前，不给高置信投注结论。
- 低比分比赛里，一次反击、定位球、门将高光或 VAR 就足以改写结果，仓位必须下调。

## 对后续工作的要求

1. 复盘阶段必须写“原预测哪里错”，不能只写赛果。
2. 每场投注建议必须有“不买/跳过”选项。
3. 子线程或子代理的事实如果互相冲突，主线程必须二次核验后再落文件。
4. 对用户造成实际损失的场次，要单独写责任复盘和流程修正。
5. 下一轮 G/H 组预测必须显式引用本文件的闸门规则。
