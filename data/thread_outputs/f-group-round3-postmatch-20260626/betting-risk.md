# F组第三轮赛后购彩复盘

先说关键结论：F组并不是“方向全错”。当前旧方案 JSON 里，`058 日本(-1) RQ away / 瑞典+1方向` 是命中的，`057 突尼斯(+2) RQ draw / 荷兰正好赢2球` 也是命中的。若用户实际“一张都没中”，需要核对真实出票是否包含这些单关或娱乐项；很多命中腿被 E组、D组或060等其他腿串关拖死，不能把 F组简单归为全错。

重点纠偏：

- 057 的 `1-3` 不是让负。突尼斯受让2球后是 `3-3`，只能按让平结算；只有荷兰赢3球及以上才是 `突尼斯+2 让负`。
- 058 的 `1-1` 对日本 `-1` 来说是让负。日本必须赢2球及以上才是让胜；赢1球是让平；不胜就是让负。
- 项目未核到 057 普通SPF可售赔率，所以 057 不能按普通客胜去复盘返奖。
- 本项目旧方案未记录官方体彩总进球/大小即时赔率；总进球和大小只用于赛果理解，不计返奖。

## 2. 057 突尼斯 1-3 荷兰：哪里命中，哪里失败

### 命中的方向

- `057 突尼斯(+2) RQ draw / 荷兰正好赢2球`：命中。旧方案中 B2、A1、A3 的娱乐项使用了这个方向。
- 荷兰胜负方向：赛果层面命中，但项目记录普通SPF未开售，旧方案没有可核官方SPF客胜票面，不能补算为中奖。
- 总进球4：若有人另买总进球4或大于2.5方向，赛果层面匹配；但旧方案没有总进球官方赔率，不计入本次旧方案命中。

### 失败的方向

- `057 突尼斯(+2) RQ home`：失败。旧方案 B1、B3 娱乐项，A2主结构中出现。实际输2球是让平，不是让胜。
- `057 突尼斯(+2) RQ away / 荷兰赢3球及以上`：失败。实际荷兰只赢2球。
- 荷兰大胜比分，如 0-3、0-4：失败。实际是 1-3，突尼斯进球使比分单断掉。

### 复盘判断

这场真正的票型教训是：看对荷兰赢，不等于买对让球。`+2` 盘的三档必须分清，输2球落在让平，不落在让胜或让负。

## 5. 为什么用户可能仍然一张都没中

1. 命中腿没有独立出票  
当前 JSON 的 C1/C2 有 `058 RQ away` 独立项。如果用户实际没有买这类单关，或出票时改成日本胜/让平，则不会命中。

2. 命中腿被其他场拖死  
不少含 058 的票同时串了 056、059、060；含 057让平的票多在娱乐仓。只要另一腿错误，整张串关仍失败。

3. 057 的“荷兰赢球”被错误转换成盘口  
荷兰赢球是对的，但如果买的是突尼斯+2让胜或荷兰赢3球及以上，就和实际 1-3 不一致。

4. 058 的“日本优势”被错误转换成日本胜或日本赢1球  
模型和红队都保留了平局尾部。实际 1-1 说明日本优势不等于SPF主胜，更不等于日本-1让平/让胜。

5. 理论返奖表容易掩盖断点  
4x11、4x4、2串1看起来覆盖更多，但每一张都受断点影响。F组局部命中不能自动抵消 E组或其他组的错误。

## 6. 以后 F 类场景风控规则

1. 让球胜平负必须写结算区间  
`+2`：不输或只输1球是让胜，正好输2球是让平，输3球及以上是让负。  
`-1`：主队赢2球及以上是让胜，赢1球是让平，不胜是让负。

2. 不把优势队写成穿盘队  
日本这类“实力和市场占优但平局可接受”的球队，不能从 SPF 低赔直接跳到让球或比分。

3. 普通SPF未开售时不补脑客胜  
057 的普通SPF未开售，只能复盘已知让球盘；不能赛后拿“荷兰赢了”去补算不存在的官方普通胜负票。

4. 同组同时开球要降低串关相关性  
057 和 058 会被实时比分互相影响，尤其是 60分钟后。不要用同组两场互相依赖的动机脚本强行拼高返奖。

5. 命中腿与中奖票分开审计  
赛后先核对每张实际出票，而不是只看方案表。单腿命中、串关失败、未出票、临场改票是四种不同问题。

6. 娱乐仓不能承担救票功能  
057让平这类高赔率方向即使命中，也常在娱乐仓且金额小；不能把它设计成主结构的保护。

## 7. 给主线程的收口结论

- 057 实际是 `突尼斯+2 让平`，不是让负；买让平命中，买让胜或让负都失败。
- 058 实际是 SPF平、`日本-1 让负`；买 058 RQ away 命中，买日本胜或日本正好赢1球失败。
- F组有命中腿，但若用户一张未中，应优先核对实际出票和串关绑定，而不是把赛后原因简化成“F组全错”。
