# 2026 世界杯分析项目

这是一个围绕 2026 FIFA World Cup Canada/Mexico/USA 的本地分析工程，用来持续整理球队、球员、赛程、赔率、战术、模型预测和赛后复盘。项目目标不是只给一句赛果判断，而是把每个预测拆成可核验的数据事实、战术判断、模型概率、红队风险和赛后修正记录，让后续分析越来越稳。

## 项目目标

- 建立 48 支球队的中文球队档案、成员表和动态状态库。
- 为每场比赛沉淀赛前预测、赔率/市场信号、风险校验和赛后复盘。
- 用统一数据格式服务模型预测，包括胜平负、比分分布、小组出线和淘汰赛路径。
- 把赛后表现、伤停、评分、角色变化回写到球员状态，持续校准后续预测。
- 对投注相关内容保持克制，只输出概率、风险和讨论性方案，不承诺收益。

## 分析工作流

本项目按“先事实、再判断、后预测、再复盘”的顺序推进。

1. 数据采集：整理球队、球员、伤停、赛程、赔率、评分和来源记录。
2. 战术分析：基于事实包判断阵型、压迫、转换、定位球、教练倾向和对位问题。
3. 模型预测：把实力、状态、战术、赔率和环境因素转成概率与比分分布。
4. 红队校验：专门找风险，包括热门过热、伤停误判、盘口诱导、样本偏差和模型过度自信。
5. 输出与复盘：写入本地 Markdown/JSON，赛后更新球员状态、比赛评分和模型备注。

## 数据原则

- 动态信息必须带来源和采集时间，尤其是伤停、停赛、首发、赔率和天气。
- 来源冲突时保留冲突，不强行合并成单一结论。
- 球员成员表优先保证中文名、位置、年龄、俱乐部中文名、身价、技术特点、状态值、评分、伤停和来源记录完整。
- 长报告和重要结论优先沉淀到本地 Markdown/JSON；聊天中可按用户需要展开，不再强制只输出摘要。
- 任务范围按用户当前要求推进；小组/轮次拆分只是组织方式，不再作为强制规则。

## 投注边界

投注相关内容只作为概率讨论和风险控制参考。没有赛前官方首发、最新伤停、官方竞彩赔率、盘口快照和红队校验时，投注状态应保持 `hold`、`discussion_only` 或 `hold_not_executable`。禁止使用“稳赚”“必红”“包中”等确定性表达。

## 常用校验

PowerShell 示例：

```powershell
# A组成员表完整性校验
python scripts\validate_a_group_rosters.py

# Python 脚本语法检查
python -m py_compile scripts\validate_a_group_rosters.py

# 检查 JSON 是否可解析
Get-ChildItem -Recurse -Filter *.json |
  ForEach-Object {
    Get-Content -Raw -Encoding UTF8 $_.FullName | ConvertFrom-Json | Out-Null
  }
```

项目进度、任务闭环和关键校验结论记录在 `线程状态.md`，它是继续分析时优先查看的状态账本。
