## 赔率采集规则

购彩建议前必须补采或复核中国体彩同源/近同源赔率：

1. 优先中国足彩网竞彩页；其次新浪体育/新浪彩票交叉验证。
2. 记录来源、抓取时间、场次编号、对阵、玩法、销售状态、SPF、RQ。
3. 明确哪些玩法未开售、停售、缺失或不同源。
4. 同一方案中尽量使用同源赔率；混源时必须标注。
5. 将赔率转为隐含概率，并在同一玩法内去水。
6. 对照模型概率计算模型-市场差，不得只按胜率高低推荐。

## 模型一致性规则

每个投注项必须回扣模型输出，并列明“模型一致性/用途”。

分类统一为：

- `顺模型`：与模型主方向、比分簇、让球真实比分条件或大小球方向一致。
- `赔率价值覆盖`：不一定是最高胜率，但去水后赔率补偿充分，且模型-市场差为正或有明确组合覆盖价值。
- `小额对冲`：与模型主方向不一致或只覆盖爆冷/平局路径，只能低仓位使用。
- `冲突`：与模型核心判断相反，默认不得进入主仓；如保留，必须重构方案并限额。

主仓只能使用 `顺模型` 或明确的 `赔率价值覆盖`。逆模型腿只能标为 `小额对冲`，不能写成稳健、保险、主推、容错或最低回款来源。

让球选项必须先翻译成真实比分条件：

- `乌拉圭(+1)胜` = 乌拉圭真实胜或平。
- `乌拉圭(+1)平` = 西班牙真实赢 1 球。
- `乌拉圭(+1)负` = 西班牙真实赢 2 球或以上。

如果模型主方向是西班牙胜/不败或西班牙赢球概率更高，则 `乌拉圭(+1)胜` 不是顺模型主仓，只能作为小额爆冷/保平对冲。

## 红队风险校验

输出任何多票方案前必须检查：

- 相关性：是否多张票绑定同一比分路径或同一冷门路径。
- 单场暴露：任意单场总敞口原则上不超过总本金 30%。
- 双核心暴露：任意两场核心焦点比赛总敞口原则上不超过总本金 45%。
- 双窄路径：双平、双 1 球、双 2+ 等窄路径合计不超过 10-15%。
- 路径覆盖矩阵：至少覆盖 A 平+B 胜、A 胜+B 平、A 1 球+B 2+、A 2++B 1 球、双胜、双平、热门不穿盘等路径。
- 最大理论回报：只能基于互相兼容、可同时发生的票据结果计算。
- 最低保险/关键场景回款：必须区分真实兜底和“若若干外围票同时命中”的上限。
- 低赔陷阱：强队低 SP 不自动等于主仓锚点。
- 爆冷/平局杀伤：若某个平局或热门不穿盘路径会击穿大部分主仓，必须降级或重构。

用户指出相关性风险时，先承认相关性成立，再重构组合树并重新计算赔率、回款和覆盖面；禁止只补一票、加一腿或替换同类腿做补丁式修补。

## 输出模板

长内容必须写入 `K:\worldcup\赔率与投注\` 下 Markdown 文件，聊天里只给路径和简表。

每张票至少包含：

| 字段 | 要求 |
| --- | --- |
| 票号/方案 | 独立方案名称，标注保守容错/均衡组合/进取组合 |
| 场次 | 中国体彩场次编号、对阵 |
| 玩法 | SPF、RQ、比分、半全场等 |
| 选项 | 体彩选项与真实比分条件 |
| 金额 | 单票金额与主仓/娱乐仓归属 |
| 赔率 | 单腿赔率与串关赔率乘积 |
| 最大理论回报 | 中奖金额上限 |
| 理论净回报 | 最大理论回报减本金 |
| 命中条件 | 必须同时成立的赛果路径 |
| 模型一致性/用途 | `顺模型` / `赔率价值覆盖` / `小额对冲` / `冲突`，附模型依据 |
| 关键风险 | 相关性、单场暴露、低赔、未开售、伤停/首发不确定 |

多方案默认结构：2 套保守容错、3 套均衡组合、3 套进取组合；单套预算默认 100 元，其中主仓 70 元、娱乐仓 30 元。不同方案是备选模板，不是叠加执行。
