# 模型优化执行方案（基于当前命中率回测）

## 目标

把当前“赛前分析 + 专家修正”的预测方式推进到更稳定的量化闭环。当前基线为：

| 指标 | 当前结果 |
| --- | ---: |
| 胜平负方向 | 15/32，46.9% |
| 前三比分命中（全样本） | 7/32，21.9% |
| 前三比分命中（明确给出至少3个候选） | 7/18，38.9% |
| 首选比分精确命中 | 4/32，12.5% |

本轮优化不追求“立刻神准”，而是先消灭口径混乱、尾部低估和赛后不回写这三类系统性问题。

## 多线程执行分工

| 线程 | 任务 | 产物 |
| --- | --- | --- |
| 数据采集 | 建立预测-赛果链接表，审计缺失字段、top5 缺口、player_state 回写缺口 | `data/packets/model_optimization/backtest-data-quality-audit-20260622.json` |
| 战术 | 把失误类型转成可量化战术波动因子 | `data/packets/model_optimization/tactical-volatility-factors-20260622.json` |
| 模型 | 给出 draw floor、事件波动、热门大胜尾部、市场拆分校准参数 | `data/outputs/model_backtests/model-parameter-update-20260622.json` |
| 红队 | 审查是否过拟合、重复计权、样本太小、盘口误用 | `data/reviews/red_team/model-optimization-red-team-20260622.json` |
| 竞彩 | 把新模型变化转成更严格的竞彩风险闸门，不给可执行购买建议 | `data/outputs/betting_risk/model-optimization-betting-gates-20260622.json` |

## 必改预测合同

后续每场预测必须同时输出：

- 数据包与输入缺口。
- 因子输入表与权重。
- xG。
- 0-0 到 5-5 泊松矩阵和 tail。
- 胜平负、大小球、top5 比分。
- 赔率隐含概率和模型差值。
- 红队状态。
- 最终概率。
- 赛后回测字段：实际比分、胜平负是否命中、top3 是否命中、Brier/log-loss、xG/总进球误差、参数更新。

## 参数优化方向

1. `draw_floor_dynamic`
   - 小组赛前两轮、双方拿分可接受、低节奏、门将/低位防守证据强时，提高平局下限。
   - 防止热门名气把 0-0、1-1 尾部压得过低。

2. `event_volatility_layer`
   - 单独处理红牌、早球、门将超常、定位球连续兑现、心理/纪律崩盘。
   - 这些不是普通 xG 线性调整，而是改变比分分布尾部。

3. `favorite_ceiling_tail`
   - 强队方向命中但大胜尾部偏低时，不能只给 2-0/2-1。
   - 当早球、对手红牌、阵地压制和替补强度同时成立时，扩展 4+ 球尾部。

4. `market_calibration_split`
   - 市场只拆成两个信号：方向信号和尾部风险信号。
   - 不能因为深热盘口就直接吞掉本地平局/冷门尾部。

5. `single_score_vs_aggregate_1x2`
   - 最高概率单比分可以是 1-1，但三项合计概率可能仍偏一方胜。
   - 后续所有报告必须分开写“最高单比分”和“胜平负第一方向”。

## 执行门槛

- 红队未完成前，模型优化只标记 `pending_review`。
- 红队给 `revise` 时，主线程必须修正参数说明。
- 竞彩线程只能输出 `discussion_only` 或 `hold`。
- 每次完赛复盘必须同步更新球员状态表和成员表，否则复盘状态只能是 `partial`。
