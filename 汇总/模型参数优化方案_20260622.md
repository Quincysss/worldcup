## draw_floor_dynamic

用途：防止低事件战略平局被热门名气或市场热度压得过低。

触发条件至少满足 3 条：

- 小组赛前两轮，双方拿分可接受，或弱势方首要目标是生存。
- 合计 final xG 在 1.6 到 2.7。
- 赛前热门胜率在 42% 到 64%，不是绝对碾压。
- 至少一队有低位/中位防守、门将状态或禁区保护证据。
- 市场热门概率比模型高 8 个点以上。
- 红队指出热门转换率、首发或伤停不确定。

修正范围：

| 场景 | 平局下限 | delta |
|---|---:|---:|
| normal | 0.245 | +0.02 到 +0.04 |
| strong_low_event | 0.275 | +0.03 到 +0.05 |
| extreme_zero_inflation | 0.300 | +0.04 到 +0.06 |

战术对齐：`draw_floor_tactical_triggers` magnitude=0.35，支持 normal/strong_low_event 档位；若无 T-75m 首发，只能作为比分分布重分配，不直接大幅改 1X2 headline。

比分矩阵处理：新增平局质量优先分配到 0-0、1-1、2-2，默认比例为 30% / 55% / 15%。若合计 xG 高于 2.6，则降低 0-0 比例，提高 2-2 比例。

反例：热门 xG >= 2.0 且对手 xG <= 0.65；弱队赛前有红牌/停赛/防线伤病；热门定位球和高点错位极强；双方都不能接受平局。

## event_volatility_layer

用途：把红牌、早球、门将超常、定位球连发、纪律崩盘和后程体能崩盘从普通 xG 线性调整中拆出来，单独改变比分尾部。

触发条件：

- 赛前纪律风险高，或已有红牌/高犯规样本。
- 高压对弱出球，或低位队被迫长时间防守。
- 门将状态极端，或替补门将/伤退风险。
- 定位球错位明显。
- 最近比赛出现 70 分钟后崩盘或必须追分脚本。

尾部调整：

| 波动等级 | event_volatility_index | tail_mass_shift |
|---|---|---:|
| low | 0.00-0.25 | 0.00 |
| medium | 0.25-0.55 | 0.015-0.030 |
| high | 0.55-0.80 | 0.030-0.055 |
| extreme | 0.80-1.00 | 0.055-0.080 |

战术对齐：`event_volatility_triggers` magnitude=0.45，属于中高波动输入；模型层 extreme 档尾部转移上限收紧到 0.060，等待红队确认后才可放宽。

调整方式：

- 纪律/崩盘型：从中位比分转向一边倒 3-0、4-0、4-1、5-1。
- 双方开放型：提高 2-2、3-2、2-3、3-3。
- 门将超常型：提高热门被零封或低进球比分，如 0-0、0-1、1-0。

反例：稳定低位、低牌风险、无压迫错位；强队领先后明显降速；该风险已在伤停或 lineup xG 中完全计入。

## favorite_ceiling_tail

用途：强队方向判断正确时，不再只挤在 2-0/2-1，要给 4+ 球尾部合理质量。

触发条件至少满足 3 条：

- 热门赛前胜率 >= 62%。
- 热门 xG >= 1.85，对手 xG <= 0.85。
- 热门有替补强度、后程进攻或定位球优势。
- 对手低位被打穿后追分能力弱。
- 市场方向信号、战术错位和本地模型同向。
- 对手纪律/体能风险存在。

尾部扩展方式：

- 默认转移 0.02 到 0.05 质量，最大不超过 0.06。
- 质量来源：热门一球小胜、低战术支撑的平局、弱队 2+ 球尾部。
- 目标比分：3-0、4-0、4-1、5-0、5-1。

战术对齐：`favorite_ceiling_tail_triggers` magnitude=0.40，支持在强热门 + 早球/替补强度/纪律崩盘/宽中双通道同时出现时启用，但 exact-score 质量转移仍不超过 0.06。

反例：热门概率主要来自市场热度而本地 xG 低于 1.65；热门核心有分钟限制；对手门将/低位证据强；热门领先后会主动降速。

## scoreline_diversity_layer

用途：修正相近球队预测里 `1-1` 过度集中。`1-1` 可以是合理的最高单比分，但不能反复成为实力接近比赛的唯一叙事。

触发条件：

- 双方内部实力差小于 8 分。
- final xG 差小于 0.35。
- 总 xG 位于 2.2-3.2。
- 原始泊松中 `1-1` 领先附近比分不足 2.5 个百分点。
- 战术层没有明确证据证明比赛必然低事件。

输出要求：

- 保留 raw Poisson Top5。
- 新增 `scoreline_diversity_layer` 与 `scoreline_clusters`。
- 至少拆分低事件平局簇、一球差均衡簇、开放平局簇、热门边际簇、弱势边际簇和波动尾部簇。
- 如果 `1-1` 是 Top1，但 1X2 第一方向是 45-48% 以下的小胜，发布为“双路径”：平局核心 + 一球差倾向。
- 如果 total xG 高于 2.8 且 BTTS 高于 58%，必须显式写出 `2-1 / 1-2 / 2-2` 相关簇。

执行状态：2026-06-28 起进入 active shadow mode，详见 `data/outputs/model_backtests/scoreline-diversity-calibration-20260628.json` 与 `汇总/比分簇多样性修正_20260628.md`。

## market_calibration_split

市场只做校准信号，不做事实。

| 组件 | 含义 | 默认权重 | 公开碎片上限 | 官方同源上限 |
|---|---|---:|---:|---:|
| market_direction_signal | 1X2 方向支持 | 0.04 | 0.06 | 0.10 |
| market_tail_signal | 大小球/让球所隐含的尾部 | 0.02 | 0.04 | 0.07 |

规则：

- 如果市场热门比模型高 12 个点以上，但模型有转换率/低位防守风险，不直接抬热门，标记 `favorite_heat_conflict`。
- 如果市场、xG、战术错位三者同向，允许小幅触发 `favorite_ceiling_tail`。
- 只有 moneyline 时，不推断大小球或大胜尾部。
- 公开文章/赔率片段的总市场影响上限为 0.06。

## prediction_contract_v2

每场预测必须输出：

- match_metadata、input_readiness。
- factor_inputs、factor_weights。
- team_strength_score、attack_score、defense_score。
- player_state_adjustment、injury_adjustment、tactical_matchup_adjustment、schedule_environment_adjustment、market_adjustment。
- expected_goals。
- 0-0 到 5-5 的 poisson_score_matrix 与 tail_probability。
- probabilities_1x2、totals_probabilities。
- top_scorelines top5。
- odds_implied_probability、model_market_delta。
- red_team_status、final_probabilities。
- quality_notes、sensitivity。
- postmatch_error 预留字段。

报告规则：

- 最高单比分和胜平负第一方向必须分开写。
- top5 必须排序，top3 可从 top5 派生用于回测。
- 使用赔率必须写来源、captured_at、overround 和归一化概率。
- 官方或同源赔率缺失时，赔率字段设 null 并写 `not_available_yet`。
- T-75m 首发缺失时，任何竞彩/投注相关输出只能 `discussion_only_hold`。

## 回测指标 v2

| 指标 | 必填字段 | 聚合方式 |
|---|---|---|
| WDL | actual_1x2、model_1x2_first_direction、wdl_hit | accuracy、probability bucket calibration |
| top3 | top3_scorelines、actual_score、top3_score_hit | all sample / available sample |
| Brier | 1X2 概率与实际 one-hot | mean_brier、按组/概率桶 |
| log-loss | -ln(actual outcome probability) | mean_log_loss、outlier list |
| xG error | team xG - actual goals | MAE、favorite/underdog bias |
| total-goal error | total xG vs actual total | mean_total_goal_error、over/under direction |

可选指标：goal_difference_error、market_delta_error、tail_event_hit_rate、draw_floor_trigger_precision、favorite_ceiling_tail_trigger_precision。

## 执行顺序

1. 先在所有新预测中执行 `prediction_contract_v2`。
2. 未来 12 到 20 场保留 base probability 与 v2-adjusted probability 双轨，做 shadow backtest。
3. 红队审查过拟合、重复计权和市场误用。
4. 每场完赛后写 WDL、top3、Brier、log-loss、xG error、total-goal error 与参数触发结果。
