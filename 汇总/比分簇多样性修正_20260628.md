# 比分簇多样性修正

更新时间：2026-06-28  
状态：active shadow mode  
关联 JSON：`data/outputs/model_backtests/scoreline-diversity-calibration-20260628.json`

## 问题

用户指出：只要双方实力接近，模型经常把比分预测成 `1-1`。这个判断成立。

根因不是 `1-1` 一定错，而是独立泊松在双方 xG 接近、总 xG 约 2.2-3.2 时，`1-1` 很容易成为最高单一比分。单个比分只是分布里的一个格子，如果报告只写 `1-1`，会掩盖周边的 `2-1`、`1-2`、`0-0`、`2-2` 和高波动尾部。

## 新规则：scoreline_diversity_layer

触发条件：

- 双方内部实力差小于 8 分。
- final xG 差小于 0.35。
- 总 xG 位于 2.2-3.2。
- 原始泊松中 `1-1` 领先附近比分不足 2.5 个百分点。
- 战术层没有明确证据证明比赛必然低事件。

触发后，预测文件必须同时输出：

| 比分簇 | 常见比分 | 用途 |
|---|---|---|
| 低事件平局簇 | 0-0、1-1 | 保留慢节奏、互相试探、淘汰赛谨慎 |
| 一球差均衡簇 | 1-0、0-1、2-1、1-2 | 表达小优势和定位球/转换边际 |
| 开放平局簇 | 2-2、3-3 | 表达双方都能进球但难拉开 |
| 热门边际簇 | 2-1、2-0、1-0 | 表达一方略强但不碾压 |
| 弱势边际簇 | 1-2、0-1、1-0 | 表达冷门一球路径 |
| 波动尾部簇 | 3-2、2-3、3-1、1-3 | 表达早球、红牌、追分、门将波动 |

## 发布口径

- 不删除 `1-1`，但不能把它当成整场唯一叙事。
- 如果 `1-1` 是 Top1，但 1X2 第一方向是 45-48% 以下的小胜，必须写成“双路径”：平局核心 + 一球差倾向。
- 如果 total xG 高于 2.8 且 BTTS 高于 58%，即使 Top1 仍是 `1-1`，也必须把 `2-1 / 1-2 / 2-2` 簇写在结论里。
- 精确 Top3 继续保留，用于回测；但对用户展示时，优先解释比分簇。

## 对当前预测的影响

当前 M81-M84 已完成正式文档，不直接改概率。后续如果用户要求重生成，或 T-75 官方首发到位，模型线程必须按这个规则重跑并补充 `scoreline_diversity_layer` 与 `scoreline_clusters`。

## 红队检查点

红队需要专门检查：凡是相近球队且 Top1 为 `1-1` 的场次，是否已经输出比分簇；如果没有，应给 `revise` 或至少在 `hold` 里要求补齐。
