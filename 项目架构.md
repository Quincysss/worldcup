# 2026 美加墨世界杯预测项目架构

## 目标

本项目用于逐队、逐场分析 2026 FIFA World Cup，最终输出：

- 每场比赛比分倾向、胜平负判断、大小球/盘口倾向和风险提示
- 小组出线形势与概率
- 淘汰赛路径与冠军预测
- 球迷友好的中文聊天分析
- 可追溯的本地文件产物

## 线程分工

1. 主线程
   - 接收用户需求
   - 拆解任务并分发给专题线程
   - 读取专题线程和本地文件结果
   - 做最终判断与聊天回复

2. 数据采集 agent
   - 采集球队、球员、赛程、伤病、停赛、赔率、天气、排名等事实数据
   - 输出结构化数据包与来源日志
   - 不做预测，不给投注建议

3. 战术/教练 agent
   - 分析阵型、压迫、转换、定位球、教练临场、关键对位
   - 输出模型可用的战术特征和球迷可转述摘要
   - 不直接预测比分或投注

4. 数据模型 agent
   - 输出胜平负概率、比分分布、小组出线、淘汰赛和冠军概率
   - 融合自建模型与赔率市场信号
   - 输出价值差、市场分歧和风险等级

5. 核验/反方 agent
   - 审查数据新鲜度、伤病误判、赔率陷阱、战术盲点、模型过度自信
   - 给出 pass / revise / hold
   - 指出需要回派给哪个 agent 修正

6. 汇总/评论员输出 agent
   - 把四个专题线程的产物整合成球迷友好的中文分析
   - 输出最终球队报告、比赛预测、赔率与投注倾向
   - 负责把可读结果落到 `队伍/`、`比赛/`、`汇总/`、`赔率与投注/`

## 调度流程

用户提出一个分析需求后，主线程按需分发：

1. 若用户要求“分析球队”，只进入 `team_profile`，不做比分、小组出线或冠军预测。
2. 数据采集 agent 先补该球队事实包和赔率/市场事实快照。
3. 战术/教练 agent 基于事实包补单队战术档案。
4. 数据模型 agent 只输出该球队模型输入简报，不输出具体比赛预测。
5. 核验/反方 agent 审查球队档案中的事实、战术、模型输入和市场事实。
6. 汇总/评论员输出 agent 写成球迷友好的单队报告。
7. 只有当用户明确要求“预测某轮比赛、某组出线或冠军”时，才进入预测阶段。

详细阶段纪律见 [工作流纪律.md](工作流纪律.md)。

## 投注表达边界

用户允许投注建议，但项目只输出理性倾向：

- 可以写：倾向、可关注、小注娱乐、等待首发确认、赔率低了不划算、风险偏高建议回避。
- 不写：稳赚、稳胆、包赢、重注、梭哈、必买。
- 每条投注倾向必须带风险点。
- 赔率、盘口、伤病和首发未更新时，投注结论必须降级或标记等待。
